              Auto-close open positions (position ledger): the server re-evaluates about every 45s
              (POSITION_AUTO_CLOSE_INTERVAL_MS optional, minimum 15s). Rules apply to all open
              rows (existing and new). Loss/profit compare unrealized $ to the trade’s initial notional at open
              (notionalUsd, with fallbacks for older rows). Real closes need the same
              wallet, relayer, and CEX gates as manual Close in P&L — otherwise auto-close logs an
              error and leaves the row open. Leave a field empty to disable that rule.
              Values are written to configs/agent.monitor.yaml under positionAutoClose when you
              click Save agent settings (the Advanced YAML editor is refreshed from disk afterward so it
              does not overwrite them with stale text).

          Scale template:
          
          
          
        

        

          
          

            Fetches wallet accounts for the wallet in Twilight wallet (step 1) and shows the row for
            TWILIGHT_ACCOUNT_INDEX (ZkOS step — default index used for real Twilight orders). NYKS / on-chain
            balances are separate from ZkOS memo balances; this snapshot is what matters for sizing the Twilight leg.
          

          

        

        

          Requires CEX / relayer setup and Allow real trading in Twilight wallet for live orders.
          Total to deploy (USD) is the combined leg notional (same kind of USD as template
          Twilight + CEX sizes): exposure before leverage splits it into margin at each venue — it is not “margin after
          leverage.” Leg shares scale proportionally vs that template.
