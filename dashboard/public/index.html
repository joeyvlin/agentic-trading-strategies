3b ZkOS account (Twilight)

        

          Real Twilight legs need a funded ZkOS trading account (an on-chain
          NYKS wallet comes first in step 1 — see skill doc
          skills/twilight-protocol-agentskill/reference-trader.md, “Wallet vs ZkOS”). You do
          not create an empty ZkOS account separately: the first Fund moves sats from
          your chain balance into ZkOS (usually index 0). Use the same wallet and password as step 1.
          Which index is used for trades comes from the index field below (and TWILIGHT_ACCOUNT_INDEX
          in .env when the field is empty). After a successful real Trade desk close, the server
          runs zkaccount transfer on that index when RELAYER_ALLOW_DASHBOARD_ZK=YES (same gate as
          manual rotate); set AUTO_ZKOS_ROTATE_AFTER_CLOSE=0 in .env to skip automation.
        

        

          
          

        

        

        

          
Account used for real Twilight trades

          

          

          

            
            
          

          

            
          

          

            Writes TWILIGHT_ACCOUNT_INDEX to .env for real strategy runs and reloads the
            process environment.
          

          

            Strategy Real runs and relayer open-trade use this index. Pick an account from the
            list output (dropdown shows io_type / tx_type from the table). Adjust the
            number if needed, then Save default index. Fund / Rotate below use their own fields (rotate
            uses “Rotate from index”). Memo means the account is locked in an order flow — use
            Advanced → Relayer: Close trade (or Cancel trade for unfilled limits), then
            after settlement Unlock settled close; if the submit failed use Unlock failed order. Then
            zkaccount transfer (Rotate) before the next open-trade on a fresh address (see skill
            order lifecycle). If auto-rotate after Trade desk close is off or blocked, use Transfer at
            100% here from the closed index yourself.

Transfer / rotate ZkOS balance

        

          zkaccount transfer (slider at 100%) moves the entire source balance
          into one new ZkOS account — you cannot target an existing index as the destination. Below 100%,
          the desk runs zkaccount split with two parts that sum to the listed source total (creates
          two new accounts). Parsed balances are best-effort from the last list output; if the relayer
          rejects the amount, try List ZkOS accounts again or use Advanced → relayer CLI.
        

        

          
          
          
        

        

          The To selector is fixed: incoming funds always open new ZkOS accounts. Use
          List ZkOS accounts after success to see new indices.
