Agentic Trading Strategies

Trading desk

Twilight relayer-cli, testnet faucet, Strategy API ideas, and a local trade journal. Dashboard binds to localhost; exchange keys stay in data/exchange-keys.json (gitignored).

Optional lock: if the server has DASHBOARD_TOKEN in its environment, enter the same value here so requests include x-dashboard-token. If the server runs without that variable, leave this empty (localhost-only is typical).

Environment

Edit the repo .env from here. The summary below shows which listed variables already have a value on disk versus which are still empty. Each category below is collapsible. Expand What "testnet" and "mainnet" presets change for the exact key/value pairs (sourced from dashboard/lib/env-catalog.mjs, aligned with the Twilight skill docs). After changing DASHBOARD_TOKEN, save then enter the same token in the header field and click Store. Secret fields stay unchanged if left blank on save.

1 Twilight wallet

        

          The dashboard runs relayer-cli on the server. Create wallet requires a
          wallet ID (any label stored in the local wallet DB, often your Twilight address after
          creation) and a password to encrypt the wallet file — same as
          relayer-cli wallet create --wallet-id … --password …. The CLI may also print a
          mnemonic once; save it offline. You can omit typing ID/password here if
          NYKS_WALLET_ID and NYKS_WALLET_PASSPHRASE are set in .env.

4 Binance / Bybit API keys

        

          Stored server-side in data/exchange-keys.json (mode 0600, not served as static
          files). Leave a field blank to keep the previous value. The agent monitor still reads
          BINANCE_* / BYBIT_* from .env — this file is for the desk and future
          wiring.
        

        

        

          
          
          
          
          
          
        

        

          
          
        

      


      

        
5 Automatic agent

        

          Edits configs/agent.monitor.yaml (YAML comments are not preserved on save). The monitor polls the
          Strategy API, enforces risk caps, then runs in simulation or real mode per
          execution.mode and AGENT_MODE in .env. Real trading also requires
          CONFIRM_REAL_TRADING=YES.
